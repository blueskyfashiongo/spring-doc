### 프라이빗 패키지 접근제한자를 실무에서 사용하는 케이스가 있는가?
  → 테스트 코드 사용할땐 쓴다
  → 라이브러리에선 많이 사용한다
  → 단, 애플리케이션 로직에선 사실 많이 보진 못했다..
    
### inner class와 nested class의 차이
  - inner class는 내부 class이다. 외부 인스턴스가 있어야 한다
  - nested class는 static으로 접근하는 클래스이다

### 클래스를 final로 선언해서 확장하지 못하는게 왜 불변으로 제공하는 방법인지 이해가 가지 않는다

### 불변의 의미는 무엇일까?
  - final로 선언하면 무조건 불변일까? → 아닌것 같다. 만약 중간에 값이 변한다면 그건 가변이다. 업데이트하는 메소드는 제공해주지만 새로운 객체가 생성된다면 그건 불변이라고 할수 있다
  - 메모리 모델 : 자바도 벤더가 여러개 있는데 명세를 봐도 구현자 마음대로 할수 있는데 그 부분도 살펴보면 좋을것 같다

### 상속용 클래스를 사전에 설계할수 있을까?
  - 상속용은 아니더라도 상속용으로 쓰면 안되는 케이스는 명확하게 알수 있을것 같다 → 이건 꼭 final로 만들어야 돼. 이건 있을수 있을 것 같다!!

### 불가피하게 불변 객체로 만들지 못하는 경우가 있을까?
  - 있는것 같다. JPA 엔티티를 사용하는 케이스일 것 같다
  - 불변이 무조건 장점만 있는건 아닌것 같아서 상황에 맞게 사용해야 할것 같다
